# **Task 7 – Regression Testing**

## **Task Description**
- Retest existing features after adding a new module.  
- Check for any potential bugs introduced by recent updates.  
- Verify that core functionalities remain unaffected.  

---

## **Test Case**

### **Test Case ID:** TC_Regression_008  
### **Title:** Test Impact of New Features on Website  

---

## **Preconditions**
- A new feature or module is added to the website.  
- Test environment is updated with the latest build.  

---

## **Test Steps**
1. Test the functionality of the newly updated areas of the website.  
2. Retest all existing major functionalities (login, homepage, search, cart, payment, etc.).  
3. Check for unexpected behaviors, UI changes, functional breaks, or side effects.  

---

## **Expected Results**
- New feature works correctly without causing any issues.  
- Existing features remain stable and fully functional.  
- No side effects or regression defects occur after updates.  

---

## **Actual Result**
- ✔ All existing features worked as expected after the new update.  
- ✔ No regression defects were found during testing.  
- ✔ New feature functioned as intended without impacting other modules.
